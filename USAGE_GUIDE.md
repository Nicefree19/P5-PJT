## 1. 시스템 개요

### 🔄 시스템이 하는 일

```
📧 Gmail 메일 수집 → 🤖 AI 분석 → 📊 Sheet 저장 → 📈 Dashboard 시각화
```

### ⏰ 자동 실행 스케줄
- **매일 오전 9시**: 자동으로 새 메일 분석
- **실시간**: Dashboard에서 최신 상황 확인
- **필요시**: 수동으로 즉시 분석 실행

### 📊 분석 결과물
- **Google Sheet**: 상세 데이터 및 분석 결과
- **Dashboard**: 실시간 시각화 및 이슈 추적
- **알림**: 긴급 상황 시 자동 알림

## 2. 첫 시작하기

### 2.1 시스템 접속

#### Google Sheet 접속
1. 브라우저에서 Google Sheets 열기
2. "P5_복합동_메일분석_DB" 시트 찾기
3. 북마크 추가 권장

#### Dashboard 접속
1. Dashboard URL 접속 (관리자에게 문의)
2. 북마크 추가
3. 모바일에서도 접속 가능

### 2.2 첫 화면 이해하기

#### Google Sheet 구성
```
📋 P5_메일_분석_DB (메인 데이터)
├── 📊 시스템_지표 (모니터링)
├── ❌ 에러_로그 (문제 추적)
└── 📈 실행로그 (실행 이력)
```

#### Dashboard 구성
```
🏗️ 메인 그리드 (12행 × 69열 기둥 배치)
├── 🔍 검색 패널 (기둥/Zone/상태 검색)
├── ⚠️ 이슈 패널 (활성 문제 목록)
└── ⚙️ 제어 패널 (필터/동기화)
```

## 3. 일상 업무 흐름

### 3.1 아침 업무 시작 (오전 9:30)

#### ✅ 체크리스트
1. **새 분석 결과 확인**
   ```
   📧 Gmail → 🤖 AI 분석 → 📊 Sheet 확인
   ```

2. **긴급 이슈 우선 처리**
   - Dashboard에서 빨간색(Critical) 이슈 확인
   - 주황색(High) 이슈 검토

3. **일일 현황 파악**
   - 전체 기둥 상태 개요
   - Zone별 진행 상황
   - 새로 발생한 문제점

#### 📱 실제 작업 순서

**1단계: Dashboard 열기**
```
1. 브라우저에서 Dashboard URL 접속
2. 로딩 완료 대기 (약 3초)
3. 전체 그리드 상태 한눈에 확인
```

**2단계: 긴급 이슈 확인**
```
1. 우측 이슈 패널에서 빨간색 배지 확인
2. Critical/High 이슈 클릭하여 상세 내용 확인
3. 관련 기둥 위치 그리드에서 확인
```

**3단계: Google Sheet 상세 확인**
```
1. Google Sheet 열기
2. 최신 행(가장 아래) 확인
3. 긴급도 컬럼으로 정렬하여 우선순위 파악
```

### 3.2 이슈 대응 프로세스

#### 🚨 긴급 이슈 발견 시

**즉시 대응 (5분 이내)**
1. **상황 파악**
   - 이슈 내용 정확히 읽기
   - 영향받는 기둥 범위 확인
   - 발생원(삼우/ENA/이앤디몰) 확인

2. **관련자 연락**
   - 해당 발생원 담당자에게 연락
   - 현장 상황 확인 요청
   - 대응 방안 논의

3. **상태 업데이트**
   - Google Sheet에서 "처리담당" 컬럼에 본인 이름 입력
   - "처리상태"를 "검토중"으로 변경
   - "메모" 컬럼에 초기 대응 내용 기록

#### 📋 일반 이슈 처리

**계획적 대응 (당일 내)**
1. **우선순위 정리**
   - High → Medium → Low 순서
   - 기둥 개수가 많은 것 우선
   - 공정에 미치는 영향도 고려

2. **체계적 검토**
   - 관련 도면 및 자료 확인
   - 유사 사례 검색 (Sheet 내 검색 활용)
   - 기술적 해결 방안 검토

3. **결과 기록**
   - "AI분석" 내용과 실제 상황 비교
   - "추천조치" 실행 여부 및 결과
   - "비고"에 최종 해결 방안 기록

### 3.3 주간/월간 업무

#### 📊 주간 리포트 (매주 금요일)

**Dashboard 활용**
1. **통계 확인**
   - 전체 기둥 수 대비 이슈 비율
   - Zone별 문제 발생 패턴
   - 이슈 해결 속도 추이

2. **트렌드 분석**
   - 반복되는 문제 유형 파악
   - 특정 발생원의 이슈 증가 추세
   - 계절적/공정별 패턴 분석

**Google Sheet 활용**
1. **데이터 필터링**
   - 최근 7일 데이터만 필터
   - 긴급도별 분류
   - 처리상태별 현황

2. **Excel 내보내기**
   - 주간 리포트용 데이터 추출
   - 차트 및 그래프 생성
   - 관리자 보고용 자료 작성

## 4. Dashboard 활용법

### 4.1 기본 화면 구성

#### 🏗️ 메인 그리드 (중앙)
```
   X1  X2  X3  ... X69
A  ⬜  ⬜  ⬜  ... ⬜   ← Zone A (FAB)
B  ⬜  ⬜  ⬜  ... ⬜
C  ⬜  ⬜  ⬜  ... ⬜
...
L  ⬜  ⬜  ⬜  ... ⬜   ← Zone C (COMPLEX)
```

**색상 의미**
- 🟢 **초록색**: 정상 상태
- 🟡 **노란색**: 주의 필요
- 🟠 **주황색**: 중요 이슈
- 🔴 **빨간색**: 긴급 이슈
- ⚫ **회색**: 해당 없음

#### 🔍 검색 패널 (상단)
```
🔍 [검색창: "A-X23" 또는 "Zone A" 입력]
```

**검색 방법**
- **기둥 검색**: `A-X23`, `B-X30` 등
- **Zone 검색**: `Zone A`, `FAB`, `CUB` 등
- **상태 검색**: `Critical`, `High` 등

#### ⚠️ 이슈 패널 (우측)
```
🚨 긴급 이슈 (2)
├── PSRC 접합부 간섭 (A-X23~X30)
└── Shop DWG 변경 요청 (C-X45)

⚠️ 중요 이슈 (5)
├── HMB 반입 지연 (B-X10~X15)
├── ...
```

### 4.2 실무 활용 시나리오

#### 시나리오 1: 특정 기둥 상태 확인

**상황**: "A-X23 기둥에 문제가 있다는 연락을 받았다"

**단계별 확인**
1. **빠른 검색**
   ```
   1. 상단 검색창에 "A-X23" 입력
   2. Enter 키 또는 검색 버튼 클릭
   3. 해당 기둥이 하이라이트됨
   ```

2. **상세 정보 확인**
   ```
   1. 하이라이트된 기둥 클릭
   2. 우측 패널에 상세 정보 표시
   3. 관련 이슈 목록 확인
   ```

3. **관련 메일 확인**
   ```
   1. "메일ID" 정보 복사
   2. Google Sheet에서 해당 행 찾기
   3. 원본 메일 내용 및 AI 분석 확인
   ```

#### 시나리오 2: Zone별 현황 파악

**상황**: "Zone A 전체 상황을 파악해야 한다"

**단계별 확인**
1. **Zone 필터링**
   ```
   1. 검색창에 "Zone A" 입력
   2. 또는 좌측 Zone 버튼 클릭
   3. Zone A 영역만 표시됨
   ```

2. **통계 확인**
   ```
   1. 하단 통계 패널 확인
   2. "Zone A: 정상 85%, 이슈 15%" 등
   3. 이슈 유형별 분포 확인
   ```

3. **문제 기둥 식별**
   ```
   1. 빨간색/주황색 기둥 위치 파악
   2. 각 기둥 클릭하여 문제 내용 확인
   3. 우선순위 순서로 대응 계획 수립
   ```

#### 시나리오 3: 새로운 이슈 등록

**상황**: "현장에서 새로운 문제를 발견했다"

**이슈 등록 과정**
1. **이슈 생성**
   ```
   1. 우측 이슈 패널에서 "+" 버튼 클릭
   2. 이슈 생성 폼 작성
   3. 영향받는 기둥 범위 선택
   ```

2. **상세 정보 입력**
   ```
   제목: [긴급] PSRC 기둥 철근 간섭 발견
   설명: X30~X35 구간에서 철근 간섭으로 콘크리트 타설 불가
   긴급도: Critical
   담당자: 홍길동
   예상 해결일: 2025-01-20
   ```

3. **관련 기둥 선택**
   ```
   1. 그리드에서 해당 기둥들 드래그 선택
   2. 또는 기둥 UID 직접 입력
   3. 저장 버튼 클릭
   ```

### 4.3 모바일 사용법

#### 📱 모바일 최적화 기능

**터치 제스처**
- **탭**: 기둥 선택
- **롱 프레스**: 상세 메뉴
- **드래그**: 다중 선택
- **핀치**: 확대/축소

**모바일 전용 UI**
- **FAB 버튼**: 우하단 플로팅 버튼
- **스와이프 메뉴**: 좌우 스와이프로 패널 전환
- **풀스크린 모드**: 세로 모드에서 최적화

## 5. 문제 상황 대처법

### 5.1 자주 발생하는 문제

#### ❌ "새 메일이 분석되지 않아요"

**원인 파악**
1. **Gmail 연결 확인**
   ```
   증상: 어제 받은 메일이 시스템에 없음
   확인: Google Sheet에서 최신 등록일시 확인
   ```

2. **키워드 매칭 확인**
   ```
   확인사항: 메일 제목에 "P5", "복합동", "PSRC" 등 키워드 포함 여부
   해결: 필요시 관리자에게 키워드 추가 요청
   ```

3. **발신자 확인**
   ```
   확인사항: 발신자가 등록된 참여자 목록에 있는지
   해결: 새로운 참여자 이메일 주소 등록 요청
   ```

**즉시 해결 방법**
```
1. 관리자에게 연락
2. 수동 분석 요청
3. 임시로 Google Sheet에 직접 입력
```

#### ❌ "Dashboard가 로딩되지 않아요"

**단계별 해결**
1. **기본 확인**
   ```
   1. 인터넷 연결 상태 확인
   2. 브라우저 새로고침 (Ctrl+F5)
   3. 다른 브라우저에서 시도
   ```

2. **캐시 정리**
   ```
   1. 브라우저 설정 → 개인정보 보호
   2. 인터넷 사용 기록 삭제
   3. 쿠키 및 캐시 삭제
   ```

3. **관리자 문의**
   ```
   연락 시 제공 정보:
   - 사용 브라우저 및 버전
   - 에러 메시지 스크린샷
   - 발생 시간
   ```

#### ❌ "데이터가 실시간으로 업데이트되지 않아요"

**동기화 확인**
1. **수동 동기화**
   ```
   1. Dashboard 상단 "🔄 지금 동기화" 버튼 클릭
   2. 동기화 완료 메시지 확인
   3. 데이터 업데이트 확인
   ```

2. **자동 동기화 설정**
   ```
   1. 설정 패널에서 "자동 동기화" 활성화
   2. 동기화 간격 확인 (기본 30초)
   3. 네트워크 상태 확인
   ```

### 5.2 긴급 상황 대응

#### 🚨 시스템 전체 장애

**즉시 대응 (5분 이내)**
1. **상황 확인**
   - Google Sheet 접속 가능 여부
   - Dashboard 접속 가능 여부
   - 다른 사용자도 같은 문제인지 확인

2. **임시 대안**
   - Google Sheet 직접 사용
   - 수동으로 메일 확인 및 기록
   - 중요 이슈는 전화/메신저로 즉시 공유

3. **관리자 연락**
   - 시스템 관리자에게 즉시 연락
   - 장애 상황 및 영향 범위 보고
   - 복구 예상 시간 확인

#### 🚨 중요 이슈 누락

**발견 시 즉시 조치**
1. **수동 등록**
   ```
   Google Sheet에 직접 입력:
   - 긴급도: Showstopper
   - 발생원: 해당 업체명
   - 제목: 명확한 문제 설명
   - 발견자: 본인 이름
   ```

2. **관련자 즉시 연락**
   - 해당 업체 담당자
   - 현장 관리자
   - 프로젝트 매니저

3. **시스템 개선 요청**
   - 누락 원인 분석 요청
   - 키워드 또는 필터 개선 요청

## 6. 고급 활용 팁

### 6.1 효율적인 데이터 활용

#### 📊 Google Sheet 고급 기능

**필터 및 정렬**
```
1. 데이터 → 필터 만들기
2. 긴급도 컬럼 클릭 → Critical만 표시
3. 수신일시 컬럼 → 최신순 정렬
```

**조건부 서식 활용**
```
이미 설정된 색상 규칙:
- Showstopper: 진한 빨간색
- Critical: 빨간색  
- High: 주황색
- Medium: 노란색
- Low: 회색
```

**피벗 테이블 생성**
```
1. 데이터 전체 선택
2. 삽입 → 피벗 테이블
3. 행: 발생원, 열: 긴급도, 값: 개수
4. 업체별 이슈 현황 한눈에 파악
```

#### 🔍 Dashboard 고급 검색

**복합 검색**
```
예시:
- "Zone A Critical" → Zone A의 Critical 이슈만
- "삼우 High" → 삼우 발생 High 이슈만
- "PSRC 접합" → PSRC 접합 관련 이슈만
```

**저장된 뷰 활용**
```
자주 사용하는 필터 조합을 저장:
1. 원하는 필터 설정
2. "뷰 저장" 버튼 클릭
3. 이름 지정 (예: "긴급이슈만")
4. 다음에 빠른 접근 가능
```

### 6.2 업무 자동화 팁

#### ⏰ 알림 설정

**이메일 알림**
```
관리자에게 요청하여 설정:
- Critical 이슈 발생 시 즉시 알림
```
